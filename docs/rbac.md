# Модель RBAC

## 1. Роли

### `user`
Обычный пользователь системы.

Что может делать:
- смотреть каталог товаров;
- создавать заказы;
- смотреть только свои заказы;
- отменять только свои заказы, если статус это позволяет;
- смотреть информацию о себе.

---

### `manager`
Сотрудник системы, который работает с товарами и заказами.

Что может делать:
- всё, что может `user`;
- создавать товары;
- редактировать товары;
- смотреть все заказы;
- менять статусы заказов.

---

### `admin`
Пользователь с полным доступом к системе.

Что может делать:
- всё, что может `manager`;
- удалять товары;
- смотреть всех пользователей;
- создавать пользователей;
- изменять пользователей;
- удалять пользователей;
- смотреть роли;
- создавать роли;
- изменять роли;
- назначать роли пользователям.

---

## 2. Разрешения

### Каталог
- `catalog:view` — просмотр каталога товаров
- `catalog:create` — создание товара
- `catalog:update` — изменение товара
- `catalog:delete` — удаление товара

### Заказы
- `orders:create` — создание заказа
- `orders:view_own` — просмотр только своих заказов
- `orders:view_all` — просмотр всех заказов
- `orders:cancel_own` — отмена только своих заказов
- `orders:update_status` — изменение статуса заказа

### Пользователи
- `users:view_self` — просмотр собственного профиля
- `users:view_all` — просмотр всех пользователей
- `users:create` — создание пользователя
- `users:update` — изменение пользователя
- `users:delete` — удаление пользователя

### Роли
- `roles:view` — просмотр ролей
- `roles:create` — создание роли
- `roles:update` — изменение роли
- `roles:assign` — назначение роли пользователю

## 4. Дополнительные правила

1. `orders:view_own` даёт доступ только к заказам самого пользователя.
2. `orders:cancel_own` даёт право отменять только свои заказы и только в допустимых статусах.
3. `orders:update_status` даёт право менять статусы заказов в рамках бизнес-логики.
4. `catalog:update` не включает в себя `catalog:delete`.
5. У `admin` полный доступ достигается через полный набор разрешений, а не через отдельную магическую проверку роли.
